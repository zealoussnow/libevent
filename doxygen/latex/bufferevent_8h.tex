\subsection{Detailed Description}
Functions for buffering data for network sending or receiving. Bufferevents are higher level than evbuffers: each has an underlying \doxyref{evbuffer}{p.}{structevbuffer} for reading and one for writing, and callbacks that are invoked under certain circumstances.

A \doxyref{bufferevent}{p.}{structbufferevent} provides input and output buffers that get filled and drained automatically. The user of a \doxyref{bufferevent}{p.}{structbufferevent} no longer deals directly with the I/O, but instead is reading from input and writing to output buffers.

Once initialized, the \doxyref{bufferevent}{p.}{structbufferevent} structure can be used repeatedly with \doxyref{bufferevent\_\-enable()}{p.}{bufferevent_8h_a1b9a3412e8c8e122c745b33d46b9643e} and \doxyref{bufferevent\_\-disable()}{p.}{bufferevent_8h_aa15582fe250247aa98d4897527215727}.

When reading is enabled, the \doxyref{bufferevent}{p.}{structbufferevent} will try to read from the file descriptor onto its input buffer, and and call the read callback. When writing is enabled, the \doxyref{bufferevent}{p.}{structbufferevent} will try to write data onto its file descriptor when writing is enabled, and call the write callback when the output buffer is sufficiently drained.
